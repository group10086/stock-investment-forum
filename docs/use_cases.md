# 交互场景文档（Use Cases）

## 概述
本文档基于26个用户故事，整理出完整的交互场景（用例），覆盖用户系统、内容系统、社交系统、信息整合系统和管理运营系统五大模块。

## 一、用户系统

---

### UC-01 用户注册

| 项目 | 内容 |
|------|------|
| **用例名称** | 用户注册 |
| **参与者** | 新用户 |
| **前置条件** | 用户未登录，访问注册页面 |
| **后置条件** | 用户账号创建成功，自动登录并跳转首页 |
| **触发条件** | 用户点击"注册"按钮 |

**基本流程：**
1. 用户访问 `/register` 页面
2. 系统显示注册表单（用户名、邮箱、密码、确认密码）
3. 用户填写信息并点击"注册"
4. 系统校验：用户名唯一性、邮箱格式、密码长度≥6、两次密码一致
5. 系统对密码进行 bcrypt 加密
6. 系统创建用户记录，生成 JWT Token
7. 系统返回注册成功信息，前端存储 Token 并跳转首页

**扩展流程：**
- 4a. 用户名已存在 → 提示"用户名已被使用"
- 4b. 邮箱已被注册 → 提示"邮箱已被注册"
- 4c. 表单校验不通过 → 显示对应错误信息

### UC-02 用户登录

| 项目 | 内容 |
|------|------|
| **用例名称** | 用户登录 |
| **参与者** | 已注册用户 |
| **前置条件** | 用户未登录，访问登录页面 |
| **后置条件** | 用户登录成功，获取 JWT Token，跳转首页或原目标页面 |
| **触发条件** | 用户点击"登录"按钮 |

**基本流程：**
1. 用户访问 `/login` 页面
2. 系统显示登录表单（邮箱/用户名、密码、记住我）
3. 用户填写信息并点击"登录"
4. 系统查询用户（支持邮箱或用户名登录）
5. 系统验证密码（bcrypt）
6. 系统检查是否被禁言/封号
7. 系统生成 JWT Token（7天有效期）
8. 前端存储 Token，跳转首页（或原目标页面）

**扩展流程：**
- 4a. 用户不存在 → 提示"账号或密码错误"
- 5a. 密码错误 → 提示"账号或密码错误"
- 6a. 账号被禁言 → 提示禁言截止时间
